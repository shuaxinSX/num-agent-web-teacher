智能体训练过程说明

本项目的智能体训练并非传统意义上的大模型参数微调，而是围绕数值分析教学场景开展的应用层训练。其核心目标不是改变模型底层权重，而是通过知识组织、输入规范、提示词约束和工作流编排，提升模型在特定教学任务中的稳定性、专业性和可解释性。结合 num-agent-web 项目的实际实现，可以将这一训练过程概括为“知识库建设、数据清洗、提示词设计、工作流闭环”四个层面。

一、知识库建设：先完成领域知识建模

该项目的知识底座并不是外挂向量库，而是以结构化课程语料的方式直接沉淀在系统内部。围绕梯形法、辛普森法和牛顿-科特斯公式，项目将教学内容统一拆分为章节导语、核心公式、复化公式、误差项、推导视角、教学重点、常见易错点、课堂提问、练习建议和默认实验参数等多个字段。这种设计本质上就是知识库 schema 的构建，它把原本依赖教师经验的课程内容转化为可复用、可渲染、可计算、可调用的结构化知识单元。

从专业角度看，这一步相当于先完成了领域知识建模。系统不是简单堆积理论文本，而是将“概念讲解、公式体系、误差解释、实验案例、教学追问”整合到统一的数据结构中，使知识既能服务页面展示，又能作为智能体回答时的语义背景与教学边界。因此，该项目的知识库训练重点不是规模，而是知识颗粒度、结构完整性和教学可复用性。

二、数据清洗：保证输入可计算、上下文可控

项目中的数据清洗主要体现在两个方面。第一类是数值实验输入的清洗与标准化。系统允许用户直接输入函数表达式和积分区间，因此在进入计算流程前，会先对表达式做统一规范处理，例如将 π 转换为 pi，将中文乘号、除号、长短横线统一为程序可识别的标准符号；随后完成分词、隐式乘号补全、AST 解析与安全求值，最终生成可执行的数学函数。这一过程有效降低了输入噪声对实验计算的影响，保证了表达式可以被稳定解释和运行。

第二类是教学交互数据的清洗。系统在恢复本地历史会话时，会过滤无效消息，只保留合法的 user 与 assistant 角色及字符串内容，从而避免脏数据进入上下文。此外，项目还会对区间端点、分段数和方法约束进行校验，例如辛普森法必须满足偶数分段，3/8 牛顿-科特斯公式必须满足分段数为 3 的倍数。这说明项目已经将“数据清洗”从传统的语料预处理，延伸到了教学任务中的参数合法性治理和上下文质量控制。

在输出端，系统还会清理模型返回中的思维标签内容，避免把模型的中间推理过程直接暴露给终端用户。这体现出一个较成熟的工程思路：不仅要清洗输入数据，也要约束输出形态，使最终回答更适合教学展示。

三、提示词设计：轻提示词、强场景约束

当前项目的系统提示词本身并不复杂，而是采用了相对克制的通用角色设定，即要求模型成为“有帮助、准确、简洁的 AI 助手”。这说明该系统并不是依赖大段复杂 prompt 去强行塑造专业性，而是更多依赖前面已经建立好的课程结构与页面任务场景，来为模型提供天然边界。

进一步来看，项目在提示词训练上的关键，不在于系统提示词本身有多长，而在于它建立了多层次的任务引导机制。例如首页设置了多个 starter prompts，围绕“方法比较”“代码实现”“公式讲解”等教学常见问题进行引导；每个章节又配置了课堂提问 prompts 和 practice 练习建议，使用户的追问天然围绕课程目标展开。换句话说，这套系统采用的是“轻系统提示词 + 强领域内容组织 + 强页面任务引导”的提示词训练思路，其目标是减少回答发散，提升模型在教学场景中的聚焦能力。

四、工作流设计：形成教学闭环而非单轮问答

该项目更有价值的一点在于，它并没有把智能体局限在单次问答，而是设计成一个完整的教学工作流。前端通过统一的 agent 工厂创建智能体实例，再根据配置接入 mock、本地代理或兼容 OpenAI 协议的后端，实现模型层与页面层解耦。用户在对话面板中的输入会被组织为多轮消息上下文，系统支持流式输出、模型切换、重新生成和本地会话持久化，这些都属于智能体工作流训练的一部分。

更重要的是，项目把“模型解释”与“数值实验验证”结合在了一起。实验面板会根据用户输入的函数、区间和分段数计算近似积分值，再以高分辨率辛普森积分作为参考值，输出绝对误差和可视化结果。这样一来，智能体不再只是给出文字答案，而是能够与实验结果共同构成“知识讲解 - 数值验证 - 误差反馈 - 再次追问”的闭环。这种闭环设计显著提升了教学智能体的专业性，因为它把语言能力与可计算结果绑定在一起，使回答更可验证、更可追溯，也更适合课堂演示和学生自学。

五、专业总结

综合来看，num-agent-web 的智能体训练过程，本质上是一种面向垂直教学场景的应用层训练。它并不依赖底层模型微调，而是通过课程知识库结构化、输入输出清洗、提示词约束和交互工作流编排，构建出一个具备专业讲解能力、实验验证能力和教学引导能力的数值分析助教系统。

如果用更正式的学术语言概括，可以表述为：本项目采用“知识工程驱动”的智能体构建路径，通过结构化课程知识建模、表达式与交互数据规范化处理、提示词对齐以及多轮会话与实验反馈的工作流集成，实现了模型在数值分析教学任务中的场景适配与能力增强。其训练对象不是模型参数本身，而是模型在特定任务环境中的知识组织方式、响应边界和教学闭环能力。
